# Contributing to PaletteForge

PaletteForge is early in development.

## Commit Style

Use Conventional Commits:

```bash
feat: add new feature
fix: fix bug
docs: update documentation
refactor: reorganize code
chore: project maintenance
```

## Branch Style

Recommended branch names:

```bash
feature/live-swap-preview
fix/auto-match-mapping
docs/project-readme
```

## Development Rule

Each patch should include:

- Version number
- Patch name
- Files changed
- Testing instructions
- Git commit message
